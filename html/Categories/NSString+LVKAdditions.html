+ lvk_parameterStringFromArray:sort:

Returns a parameter string, which can optionally be alphabetized.

+ (NSString *)lvk_parameterStringFromArray:(NSArray *)array sort:(BOOL)sort

Parameters

array

The two-dimensional array to convert.
sort

Whether or not to sort the array.

Return Value

A new string instance formatted as a parameter string.

Discussion

Returns a parameter string, which can optionally be alphabetized.

The array is expected to mirror the format returned from calls to lvk_convertParameterStringToSortedArray, namely that it is a two-dimensional array with the first element being the key and the second element being an array of value strings.

– lvk_convertParameterStringToSortedArray

Returns an array with two elements, a string and another sorted array.

- (NSArray *)lvk_convertParameterStringToSortedArray

Return Value

A new two-dimensional array built from parsing the string.

Discussion

Returns an array with two elements, a string and another sorted array.

The first element will be a the key, and those strings are sorted. The second element is an array, which contains the sorted values for that key.

This matches the URL specification where it is possible to have two entries for one key. For instance, this is legit:

"foo=baz&foo=bar"

In this case, the returned array would be:

[“foo”, [“bar”, “baz”]]

If the string is simple in nature, then the outer data structure returned is identical, but the values array is empty:

[“foo”, []]
