Overview

A LVKMediaSource represents a single, playable resource.

The most basic element of a Media is an encoding, but some source mechanisms can package multiple encodings into one URL. As such, the MediaSource represents a collection of one-to-n encodings that are packaged together and playable as a single unit.

For example, Widevine streaming and HLS do automatic bit-rate switching, while Widevine offline only has a single bit rate (and, as such, a single encoding). For your convenience, all you need is the MediaSource for the set of encodings you want to play.

Note, LVKPlayerController will pick a reasonable media source if given just a media, but for more fine-grained playback control a media source can also be specified.

downloadable

Whether the source is downloadable, ex. Widevine Offline.

@property (readonly, nonatomic, getter=isDownloadable) BOOL downloadable

Discussion

Technically, a streaming format such as HLS could be downloaded & streamed from disk, but this SDK only provides support for downloading DRMprotected formats designed to be downloaded.

group

For applicable MediaSource subtypes, returns extra delineation between otherwise similar subtypes.

@property (readonly, nonatomic) NSString *group

Discussion

Some media sources, most notably all Widevine-related sources, populate this property to help developers using this SDK to differentiate between SD/HD/Offline without having to check class types or bitrate ranges.

Possible values:

  • HD (high definition streaming)
  • SD (standard definition streaming)
  • Offline (non-streaming & configurable bitrates)

localURL

The URL of the downloaded source, if applicable.

@property (readonly, nonatomic) NSURL *localURL

Return Value

If the source has been completely downloaded, returns the filesystem URL. If the source has not been downloaded, including being in-progress, returns nil.

Discussion

Tip: KVO this property to get download completion notifications.
